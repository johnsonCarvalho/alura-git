git init: Cria um repositório local
git status: mostra o status do repositório, incluindo os arquivos modificados e não comitados e o ramo de trabalho. https://git-scm.com/book/pt-br/v2/Fundamentos-de-Git-Recording-Changes-to-the-Repository
git add ou git add .: indica que o arquivo(s) estarão sob controle de versão
git commit -m "mensagem": commit os arquivos modificados ou novos no ramo
git config --local user.name "Johnson": Configura o usuário utilizado pelo git no projeto. Utilize --global para todos os projetos da máquina.
git config --local user.email "johnson@stj.jus.br": Configura o e-mail utilizado pelo git no projeto. Utilize --global para todos os projetos da máquina.

https://devhints.io/git-log
git log --oneline: visualiza todos os commit, um por linha - poucas informações.
git log -p: visualiza todos os commit. O formato em que elas são exibidas conta com a versão anterior em vermelho, e a versão modificada logo abaixo, em verde.
